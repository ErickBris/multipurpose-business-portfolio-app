Import project - top

After downloading packed file, unzip it and follow these instructions:

Open Eclipse, click File -> Import... Under Android folder, choose Existing Android Code Into Workspace and click Next. Browse for unzipped folder and choose android folder. Click OK, then click Finish. Before clicking Finish, you may rename project name by clicking BusinessPortfolio under New Project Name tab and typing your own project name. Repeat the procedure for google-play-services_lib project.

Set Up Server - top

If you don't have available public server with public IP address, you can declare data through java source code. More details in this section.

Before you run your application, you should set up server and database and insert some data. Required database: MySQL. Required HTTP server: Apache or any compatible HTTP server that supports PHP module.

First, you should create database. Remember full database name. After that, create database user, give it CRUD permisions and add that user to database. Remember full database username and password.

If you have CPanel instaled on your server, you can create new database and database user using MySQLÂ® Databases or PhpMyAdmin tool from your CPanel or you can ask your server administrator to create one.

Next 3 images show different CPanel options for 3 popular hosting providers (HostGator, BlueHost and AwardSpace, respectively):

After you create database and user, open folder where you have unzipped purchased product, then open folder named "server". You'll find there 'business_portfolio.sql' file. Open it with any text editor (preferebly use Notepad++) and find 22nd line of code, where you should change your database name that is written under apostrophe signs. Then save file and import it using Import function in PhpMyAdmin tool. Your database should be ready now.

All you now need to do is: upload "business.zip" file to PUBLIC_HTML folder on your server and extract it. You should get 'business' folder in it. Open that folder, then open 'protected' folder, then find 'config' folder and in it 'main.php' file. Open it find this part of code:

'db'=>array(
			'connectionString' => 'mysql:host=localhost;dbname=business_portfolio',
			'emulatePrepare' => true,
			'username' => 'root',
			'password' => 'password',
			'charset' => 'utf8',
			'tablePrefix' => '',
		),

Change params that are written italics with newly created database name and username and password (not the database password, but username password). Then go to yourdomain.com/business and log in with admin/admin and add new data :)

What You Must Change - top

In order for you application to work with you server database data, you need to change path to server. Open SyncDataHelper.java file and change lines that fit you server data:

public static final String URL_ROOT = "http://www.pengaja.com/business/index.php/";

public static final String URL_UPLOADS = "http://www.pengaja.com/business/uploads/";

You should change www.pengaja.com to yourdomain dot com.

Don't Have Server? - top

If you don't have server, this is not a problem. You can define every data programmatically in SyncOfflineData.java. You have an example on how to set up data when you open that file. And then, open MainActivity.java file, comment 55th and 69th line of code, and delete comment in 56th and 70th line of code. If you have changed anything, and can't find these lines of code, it should look like this (55th and 56th lines of code):

...

//private SyncData mSyncData;

private SyncOfflineData mSyncData;

...

69th and 70th lines of code:

...

//mSyncData = new SyncData(this);

mSyncData = new SyncOfflineData(this);

...

After that, just import SyncOfflineData.java (right click on that line of code).

What You Should and May Change - top

There are some thing you should change to fit your company info. You will generally change .java, .html and .xml files. If you are about to change .java files, we've made shortcuts in our code. We added TODOs on every part of code that should and could be changed. Those things that you could change are written inside brackets.

To quickly find every TODO in Eclipse project, click Window -> Show View -> Other... Find Tasks View under General folder and click OK.

Left Menu Items - top

There are 5 left menu items in this application (Portfolio (Gallery), About Us, Our Team, Clients and Contact). Every item can be excluded deleting just one line of code. If you would like to sort them differently, cut and paste any line of code before or after another.

To add new item, open 'LeftMenuItem.java' file (in 'com.csform.businessportfolio.model' package), create new public, static and final String like this:

public static final long LEFT_MENU_MY_LEFT_MENU_ITEM = 16;

Then, in 'MainActivity.java', in initLeftMenuItem() method, add new line for 'LEFT_MENU_MY_LEFT_MENU_ITEM' following the code for previous items. Then create new Fragment like one in 'com.csform.businessportfolio.fragment' package and follow the code from 'MainActivity.java' method selectItem(long tag) in order for your fragment to work on left menu drawer click.

Change Contact Information - top

To change your contact information, open ContactFragment.java in fragments package and find TODO after which you should find data to change.

To change your location on map, open MapActivity.java file, and change LatLong to fit your needs. If you have more locations, add more LatLong objects by just copy-pasting that line of code and changing data.

Change AdMob ID - top

In order for your ads to work, you have to register (or, if registered, log in) here. After registering/logging in, go here, choose Android app and fill information. When you create new site, go here, find your newly created site, hover with mouse over it and click Menage Settings. Find Publisher ID and copy it.

After that, in Eclipse, open your project, open res folder, open layout folder, open main_activity.xml file. There, find (about line 12)

ads:adUnitId="MY_AD_UNIT_ID"

Change "MY_AD_UNIT_ID" with copied Publisher ID. When you run code in emulator or on phone, your ads should start showing in few minutes.

Change Analytics Tracking ID - top

Log in with your Google account here. In the Account column, select the account from the dropdown that you want to add the property to. If you have a lot of accounts, use the search box to help you find the right one. In the dropdown in the Property column, click Create new property. Select Mobile app and fill the rest of fields.

After submiting, copy your tracking ID that is next to name of your newly created site. It should look like with UA-XXXX-Y. Copy it, open Eclipse, find res folder, find values folder and open analytics.xml and paste instead of UA-XXXX-Y.
